## Notes

1. **"Sent to which provider"** depends entirely on which model the developer selects in their IDE. If the organization restricts models to OpenAI/Microsoft fine-tuned only (via enterprise policy), data never leaves the Azure ecosystem.
2. **Content exclusion gaps**: Exclusions do not apply to Copilot CLI, coding agent, Agent mode, or Edit mode. Semantic type information from excluded files may still leak through IDE-provided context.
3. **NPI risk is contextual**: Source code itself is not NPI, but code frequently contains embedded NPI (test data, connection strings, schema definitions, business logic comments). The regulatory risk ratings above assume a financial services codebase where NPI exposure is plausible.
4. **Training prohibition is contractual, not technical**: No model provider has a documented technical mechanism preventing training on Copilot data — the protection is the ZDR agreement and contractual no-training commitment.
